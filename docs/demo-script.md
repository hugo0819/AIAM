# 演示脚本 · 5 分钟逐字稿

> 字节跳动创新挑战赛《给 AI 发通行证》参赛 Demo · 现场版口播稿
>
> 角色：**小李**（持有人 / 演示者），**Aria**（AI 行政助理 / 剧本化 Agent）。
>
> **演示前 15 秒准备**：
> 1. 打开两个浏览器窗口。窗口 A 主屏：`http://your-app/stage/s1`（演示舞台）。窗口 B 副屏：稍等 S5 出现时再打开。
> 2. 控制台另一个标签页：`http://your-app/console/audit`（审计与图谱）和 `http://your-app/console/risk`（风险中心）。
> 3. 演示前点击 **「重置环境」** 按钮（首页右上角）—— 清空数据库 + 内存吊销名单 + 重新写 seed。
> 4. 浏览器允许弹出窗口（S5 会自动开新窗口模拟手机推送）。

---

## 0. 开场（30s）

> 「大家好。今天我想回答一个问题——
>
> **当 AI Agent 真正开始替我们干活，我们拿什么管住它？**
>
> OAuth 是为人设计的，RBAC 是为应用设计的，但 Agent 是**代人行动、会调 Agent、行为非确定、上下文里夹带敏感数据**的新物种。
>
> 所以我们做了一个叫 **AI Agent Passport** 的东西——给每一个 AI，发一张数字通行证。」

**画面**：首页 Hero —— C1 / C2 / C3 / C4 四锚点卡片，slogan「可控 · 可审 · 可追责」。点击「**从 S1 开始演示**」。

---

## 1. S1 · 签发通行证（45s · 锚点 C1 · 能力令牌）

> 「这是我的 AI 助理 Aria。现在我给她签一张通行证。」
>
> 「这张通行证不是粗粒度的 token，而是一份**结构化声明**——
> 她能用哪些工具、访问哪些数据等级、单次金额上限多少、有效到什么时候，
> 全部都在这张 ES256 签名的 JWT 里，任意网关都可以离线验签。」

**操作**：在 S1 卡片上点「播放剧本」。

**画面焦点**：
- 顶部 `passport: (未签发)` 徽标变为 `passport: AbCdEf…` 绿色徽标 → 已写入 localStorage 跨场景共享
- 新窗口 / 切到「通行证」Tab，可看到一张**真实护照尺寸**（1.585:1）的卡片，正面 Aria 头像 + 状态戳 + 风险等级，**边框带流光动画**
- 点击翻转 → 背面是结构化能力清单：`expense.submit { amountMax: 5000, currency: CNY }`、时间窗、地理、数据等级

**关键台词**：
- 「这是整个系统的视觉符号——一张可见的、可签可验可吊销的数字护照。」
- 「注意背面这条 `expense.submit · amountMax 5000`——这是 capability 级的硬限。一会儿会用到。」

---

## 2. S2 · 正常调用（30s · 基础能力 · 10 步审计链）

> 「先走一个最简单的路径。让 Aria 帮我约一个明天下午 3 点的项目评审。」

**操作**：返回 stage，点 S2 → 播放剧本。

**画面焦点**：
- ScenarioRunner 中两步 invoke 依次变绿 → `decision=EXECUTED`
- 切到「审计与图谱」Tab → **事件时间线实时滚动**，每个工具调用产生 6 步事件：`RECEIVED → VERIFIED → POLICY_CHECKED → MASKED → EXECUTED → LOGGED`
- 顶部 SSE 红点常亮，未处理事件统计实时跳动

**关键台词**：
- 「网关把每一次调用拆成 6 个原子事件，每一步都可点开看到当时的上下文。」
- 「**你永远知道是谁、在什么时间、用哪张通行证、调了哪个工具、命中了哪条策略。**」

---

## 3. S3 · 数据脱敏（50s · 锚点 C2 · 数据平面管控）

> 「下一个，演示我们最重要的差异化创新——**数据平面的管控**。」
>
> 「很多 AI 安全方案到'权限通过了就放行'就停了。但 Agent 拿到数据就会进 LLM 上下文，
> **上下文里夹带 PII 是非常危险的**。所以我们的网关不止拦请求，还要拦数据。」

**操作**：S3 → 播放。这一步会拉客户名单（含手机号、身份证、邮箱）。

**画面焦点**：
- 切到「审计与图谱」时间线
- 找到刚才那条 `crm.list_customers` 的 `EXECUTED` 事件，**点开展开**
- 看到 4 个并列卡片：**入参原始 / 入参脱敏 / 出参原始 / 出参脱敏**
- 重点对比出参：
  - 原始：`phone: "13812341234"`、`idCard: "330102199001010012"`、`email: "wang@example.com"`
  - 脱敏：`phone: "138****1234"`、`idCard: "3301**********0012"`、`email: "w***@example.com"`
- 事件 note 上写着：`出参脱敏 · regex: 9 处（phone_cn, idcard_cn, email）`

**关键台词**：
- 「网关在工具返回数据之后、交给 LLM 之前，做了正则 PII 识别 + 字段级补刀。」
- 「**LLM 永远拿到的是右边那个版本，但审计日志保留左边的供追溯。**」
- 「这不是简单的中间件——这是治理。」

## 4. S4 · 越权拦截（25s · 基础能力）

> 「如果 Aria 想干超出能力的事呢？比如她突然想看同事工资。」

**操作**：S4 → 播放。

**画面焦点**：
- ScenarioRunner 步骤变红 → `expect: DENY` 满足
- 时间线出现一条 `POLICY_CHECKED · DENY` 事件，note 显示 `capability_missing · tool hr.query_salary not in capabilities`

**关键台词**：
- 「能力清单里没有就是没有——三道防线依次命中：能力匹配失败 → 策略引擎兜底 → 审计留痕。**毫秒级。**」

---

## 5. S5 · JIT 即时升级（70s · 锚点 C3）

> 「第三个创新，叫做**即时风险升级**。」
>
> 「传统思路是事先把所有权限给足，省事但风险大。我们的思路是——
> **不给多，到需要的时候当场让人来确认。**」

**操作**：S5 → 播放。这一步会提交 600 元报销，超过策略阈值 500，触发 STEP_UP。

**画面焦点**：
- ScenarioRunner 步骤进入 running 状态后**几秒不返回**——这就是网关在阻塞等审批
- **副屏窗口自动弹出**（如果浏览器允许）：一个手机推送样式的卡片
  - 玻璃质感背景 + 顶部 `STEP_UP` 黄色徽标
  - 标题「单笔报销超过 500 元，需持有人二次确认」
  - JSON 格式展示请求参数 `{ amount: 600, currency: "CNY", category: "差旅" }`
  - 底部 60 秒倒计时进度条（黄→红渐变）
  - 大号「拒绝 / 批准」按钮

**操作**：副屏点「**批准**」。

**画面焦点**：
- 副屏卡片切换为绿色 ✓ 「已批准」
- **主屏的 ScenarioRunner 步骤同步变绿** —— 同一个 invoke 调用从挂起恢复 → DLP → 工具执行 → EXECUTED
- 时间线上多了 4 条事件：`STEP_UP → ALLOW(JIT 审批通过，继续执行) → MASKED → EXECUTED`，policyHits 标记 `policy:expense-stepup`

**关键台词**：
- 「网关层用 EventBus + Promise **真正阻塞**了这次调用——不是占位、不是模拟。」
- 「我批准就继续，我拒绝就 DENY，60 秒不响应自动超时。审计上这一步会被标记为**经人工核准**。」
- 「这种'权限不是给死的，是临时申请的'思路，是 AI Agent 治理与传统 RBAC 的本质区别。」

---

## 6. S6 · 委托链派生（70s · 锚点 C4 · 信任图谱）

> 「最后一个核心创新——**信任链路的可视化**。」
>
> 「Aria 要订机票，她不直接做，而是委托一个'差旅子 Agent'。
> 这时候通行证会**派生出一张能力是父集子集的子通行证**——
> 子 Agent 永远不可能拿到比父更宽的权限。」

**操作**：S6 → 播放。

**画面焦点**：
- 第一步 `issue` 变绿，顶部出现 `derived: AbCd…` 紫色徽标
- 第二步 invoke 调用 `flight.book` —— 注意这是用**派生 JWT** 发起的（runner 自动选择）→ EXECUTED，返回 `pnr=PNR-XXXXXX`
- 切到「审计与图谱」→「**信任图谱**」Tab
- React Flow 力导向图实时绘制：
  - 第一列 蓝色节点 **User · 小李**
  - 第二列 紫色节点 **Aria** + **Travel 子 Agent**，通过 `owns` 边连到 User
  - 第三列 青色节点两张 Passport，通过 `holds` 边连到各自 Agent
  - 父子 Passport 之间一条**加粗紫色边 `derives`**
  - 第四列 绿色节点 `flight.book` 工具
  - 一条**绿色动画边 `invokes × N`** 从子 Passport 指向 flight.book

**关键台词**：
- 「派生时网关做 subset 校验：tool 必须包含、scope 必须 ⊆、amountMax 必须 ≤、dataClearance 必须 ⊆。**任何放大权限的尝试都会被拒。**」
- 「你看图里这条链路：**User → Agent → Sub-Agent → Tool → Data**。从动作回溯到人，端到端可追，责任归属一目了然。」

---

## 7. S7 · 异常告警 + 一键吊销（60s · 生命周期闭环）

> 「最后一个场景：那如果 Aria 被劫持了呢？」

**操作**：S7 → 播放。脚本会模拟「凌晨 3 点连续 3 次大量拉客户数据」。

**画面焦点**：
- 三步 invoke 都 EXECUTED，但 ScenarioRunner 的 detail 行写着 `ctxOverride { hour: 3 }`
- 侧边栏「风险中心」徽标变红，跳出数字 **1**
- 切到风险中心：
  - 顶部 4 个统计卡：CRITICAL 0 · **WARN 1** · INFO 0
  - 一张橙色边框的 `WARN · off-hours-bulk-read` 告警卡片，描述「非工作时段（3:00 左右）检测到高频读类工具调用 ×3」
  - 右侧两个按钮：「**一键吊销**」+「确认」
- **点「一键吊销」**，输入框默认填好「风险告警触发的紧急吊销」，确定

**画面焦点（接续）**：
- 通行证页面看 → 父+子两张卡都叠上斜纹 `REVOKED` 红色印章
- 时间线再看 → 接下来的两条调用都是 `VERIFIED · DENY · invalid_token (revoked)`
- 信任图谱里两个 passport 节点变红

**关键台词**：
- 「风险监控不阻塞主流程，是 fire-and-forget 的——但识别后立刻入告警库 + SSE 推送。」
- 「**吊销名单是内存级的，1ms 内对所有验签生效。派生通行证级联失效。**」
- 「这就是 AI Agent 的完整生命周期治理——**签发 → 运行 → 脱敏 → 审批 → 派生 → 监控 → 吊销**，闭环。」

## ✦ 现场 Tips

1. **绝不接真 LLM**：所有 Agent 行为走 `mock/scenarios.ts` 剧本，确保不翻车。
2. **演示前必跑「重置环境」**：清空数据库 + 内存名单。
3. **两个浏览器窗口**：S5 必须能弹副屏。
4. **录屏兜底**：提前在同样的 viewport 录一份 1080p 视频，现场翻车切播放。
5. **PPT 最后一页**：贴一张 Passport 的 JWT Payload JSON 截图，这是灵魂道具。
6. **金句**：
   - "OAuth 是给人设计的，Agent 需要新的身份模型。"
   - "权限不是一次给死的，是按需申请的。"
   - "权限过了不等于数据可以漏。"
   - "签发 → 验证 → 脱敏 → 审批 → 派生 → 监控 → 吊销，一个都不能少。"

## ✦ 应急预案

| 风险 | 兜底 |
|---|---|
| S5 弹窗被浏览器拦截 | 把 stage 页面 URL 提前在地址栏切换到 `/approve/{id}`（事件流里有 id） |
| SSE 没连上 | 时间线右上角徽标显示「未连接」时刷新页面 |
| 数据库脏了 | 首页右上角「重置环境」按钮一键清场 |
| 网络炸了 | 用录屏顶上 |
| 被问「为什么不用 OAuth」| 答：「OAuth 是为'人对应用'设计的；我们的能力令牌包含 Agent 特有维度——派生深度、运行时数据等级、JIT 升级触发器、行为基线，这些是 OAuth scope 表达不了的」 |
